## 1. 网络概述

### 什么是计算机网络

计算机网络是由两台或多台计算机通过通信线路和通信设备连接而成的系统，目的是实现资源共享和信息交换。

### 网络的基本组成

| 组件 | 说明 | 示例 |
|------|------|------|
| **计算机设备** | 网络中的终端 | PC、手机、服务器、IoT设备 |
| **网络设备** | 连接和转发数据 | 路由器、交换机、集线器 |
| **传输介质** | 数据传输载体 | 双绞线、光纤、无线信号 |
| **通信协议** | 通信规则标准 | TCP/IP、HTTP、FTP |
| **网络软件** | 控制网络运行 | 操作系统网络栈、驱动程序 |

### 网络的主要功能

1. **资源共享**：共享打印机、文件、数据库等
2. **通信交流**：电子邮件、即时通讯、视频会议
3. **分布式计算**：多台计算机协作完成复杂任务
4. **信息管理**：集中存储和管理数据
5. **娱乐休闲**：在线游戏、流媒体服务

## 2. 网络类型

### 按地理范围分类

| 类型 | 范围 | 设备 | 典型应用 |
|------|------|------|----------|
| **PAN** | 个人（<10m） | 蓝牙适配器 | 穿戴设备互联 |
| **LAN** | 局部（<1km） | 交换机、路由器 | 家庭/办公室网络 |
| **MAN** | 城市（10-100km） | 光纤、路由器 | 校园网、城域网 |
| **WAN** | 国家/全球 | 路由器、光端机 | 互联网、企业VPN |
| **GAN** | 全球 | 卫星、海底光缆 | 全球互联网骨干 |

### 局域网（LAN）

局域网是最常见的网络类型，覆盖范围有限。典型办公室 LAN 包括多台 PC 通过交换机连接，再经路由器接入互联网。

### 城域网（MAN）

城域网覆盖城市范围，通常由光纤连接：
- 校园网互联
- 城市公共 WiFi
- 有线电视网络
- 图书馆、医院互联

### 广域网（WAN）

广域网覆盖范围广，通常由电信运营商提供：

| 接入方式 | 速率 | 特点 |
|----------|------|------|
| 电话线拨号 | 56Kbps | 已基本淘汰 |
| ADSL | 1-24Mbps | 上行慢、下行快 |
| 光纤接入 | 100M-10Gbps | 速度快、稳定性好 |
| 无线 4G | 10-100Mbps | 移动性好 |
| 无线 5G | 100M-1Gbps | 高速率、低延迟 |

### 按拓扑结构分类

| 拓扑 | 结构 | 优点 | 缺点 |
|------|------|------|------|
| 总线型 | 所有设备连在同一主线 | 结构简单、成本低 | 故障影响大 |
| 星型 | 设备连向中心节点 | 易于管理、故障隔离 | 依赖中心设备 |
| 环型 | 设备连成闭环 | 结构简单 | 故障难排查 |
| 网状型 | 设备多向连接 | 高可靠性 | 成本高、结构复杂 |
| 树型 | 分层星型结构 | 扩展性好 | 根节点故障影响大 |

## 3. 网络设备

### 集线器（Hub）

工作在物理层，简单复制信号。

**特点：**
- 工作在物理层
- 无智能转发，广播所有数据
- 所有设备共享带宽
- 已基本被交换机取代

### 交换机（Switch）

工作在数据链路层，智能转发。

**特点：**
- 工作在数据链路层
- 基于 MAC 地址转发
- 每个端口独享带宽
- 支持 VLAN

**MAC 地址表示例：**

| MAC 地址 | 端口 |
|----------|------|
| 00:11:22:33:44:55 | Port 1 |
| 00:11:22:33:44:66 | Port 2 |
| 00:11:22:33:44:77 | Port 3 |

### 路由器（Router）

工作在网络层，连接不同网络。

**特点：**
- 工作在网络层
- 基于 IP 地址转发
- 连接不同网络（LAN-WAN）
- 支持路由协议
- NAT、ACL、防火墙功能

**路由表示例：**

| 目标网络 | 子网掩码 | 下一跳 | 接口 |
|----------|----------|--------|------|
| 192.168.1.0 | 255.255.255.0 | 0.0.0.0 | LAN |
| 0.0.0.0 | 0.0.0.0 | 202.96.128.1 | WAN |

### 三层交换机

结合二层交换和三层路由：

| 对比 | 二层交换机 | 三层交换机 |
|------|-----------|-----------|
| 工作层 | 数据链路层 | 网络层+数据链路层 |
| 转发依据 | MAC 地址 | IP 地址 |
| VLAN 间通信 | 需路由器 | 内置路由功能 |
| 适用场景 | 接入层 | 汇聚层、核心层 |

### 防火墙（Firewall）

网络安全设备：

| 类型 | 说明 | 部署位置 |
|------|------|----------|
| 包过滤防火墙 | 检查数据包头部 | 网络边缘 |
| 状态检测防火墙 | 跟踪连接状态 | 网络边缘 |
| 应用层防火墙 | 深度检查内容 | 服务器前 |
| 云防火墙 | 云端防护 | 云服务边界 |

### 无线接入点（AP）

提供无线网络接入，现代 AP 可通过 PoE 供电，支持 802.11ac/ax 协议。

## 4. 网络拓扑

### 总线型拓扑

所有设备连接到一条主干线缆，两端需要终端电阻。

**特点：**
- 结构简单，成本低
- 总线故障全网瘫痪
- 冲突域大

### 星型拓扑

所有设备连接到一个中心节点（通常是交换机）。

**特点：**
- 易于管理和扩展
- 单点故障不影响其他节点
- 中心节点是瓶颈

### 环型拓扑

设备连接成闭环，数据沿一个方向流动。

**特点：**
- 单向传输，效率高
- 故障难定位
- 常用于令牌环网络

### 网状拓扑

设备之间有多条连接路径。

**特点：**
- 高可靠性
- 多路径冗余
- 成本高，配置复杂

## 5. OSI 参考模型

### 七层详解

| 层 | 名称 | 职责 | PDU | 设备/协议 |
|----|------|------|-----|-----------|
| 7 | 应用层 | 用户接口服务 | Data | HTTP, FTP, SMTP |
| 6 | 表示层 | 数据格式转换 | Data | SSL, TLS, ASCII |
| 5 | 会话层 | 会话管理 | Data | NetBIOS, RPC |
| 4 | 传输层 | 端到端传输 | Segment | TCP, UDP |
| 3 | 网络层 | IP 寻址路由 | Packet | Router, IP |
| 2 | 数据链路层 | 帧传输 | Frame | Switch, MAC |
| 1 | 物理层 | 比特传输 | Bit | Hub, 光纤 |

### 数据封装流程

```
发送端                                    接收端
┌─────────┐                              ┌─────────┐
│ 应用层   │ ←──── 应用数据 ────→         │ 应用层   │
└────┬────┘                              └────┬────┘
     │ 数据                                    │ 数据
┌────┴────┐                              ┌────┴────┐
│ 传输层   │ ←─→ Segment ←─── →         │ 传输层   │
└────┬────┘                              └────┬────┘
     │                                        │
┌────┴────┐                              ┌────┴────┐
│ 网络层   │ ←─→ Packet ←─── →           │ 网络层   │
└────┬────┘                              └────┬────┘
     │                                        │
┌────┴────┐                              ┌────┴────┐
│ 数据链路 │ ←─→ Frame ←─── →            │ 数据链路 │
└────┬────┘                              └────┬────┘
     │                                        │
┌────┴────┐                              ┌────┴────┐
│ 物理层   │ ←─→ Bit ←─── →              │ 物理层   │
└─────────┘                              └─────────┘
```

## 7. IP 地址详解

### IPv4 地址结构

32 位二进制，分为 4 个字节：

```
11000000 . 10101000 . 00000001 . 00000001
   192    .    168    .     1    .     1
```

### 地址分类

| 类 | 范围 | 网络位数 | 主机位数 | 默认掩码 | 私有地址 |
|----|------|----------|----------|----------|----------|
| A | 1-126 | 8 | 24 | /8 | 10.0.0.0/8 |
| B | 128-191 | 16 | 16 | /16 | 172.16.0.0/12 |
| C | 192-223 | 24 | 8 | /24 | 192.168.0.0/16 |
| D | 224-239 | - | - | - | 组播 |
| E | 240-255 | - | - | - | 保留 |

### 子网掩码

| CIDR | 子网掩码 | 可用主机数 |
|------|----------|------------|
| /24 | 255.255.255.0 | 254 |
| /25 | 255.255.255.128 | 126 |
| /26 | 255.255.255.192 | 62 |
| /27 | 255.255.255.224 | 30 |
| /28 | 255.255.255.240 | 14 |
| /29 | 255.255.255.248 | 6 |
| /30 | 255.255.255.252 | 2 |

### 子网划分计算

给定网络 192.168.1.0/24，需要划分为 4 个子网：

**计算方法：**
- 借位：需要 2 位（2^2 = 4）
- 新掩码：/26（24 + 2）
- 每个子网可用主机：2^(32-26) - 2 = 62

**划分子网：**

| 子网 | 网络地址 | 可用范围 | 广播地址 |
|------|----------|----------|----------|
| 1 | 192.168.1.0 | 192.168.1.1 - 192.168.1.62 | 192.168.1.63 |
| 2 | 192.168.1.64 | 192.168.1.65 - 192.168.1.126 | 192.168.1.127 |
| 3 | 192.168.1.128 | 192.168.1.129 - 192.168.1.190 | 192.168.1.191 |
| 4 | 192.168.1.192 | 192.168.1.193 - 192.168.1.254 | 192.168.1.255 |

### IPv6 基础

128 位地址，冒号十六进制表示：

```
完整:  2001:0db8:85a3:0000:0000:8a2e:0370:7334
压缩:  2001:0db8:85a3::8a2e:0370:7334

类型前缀:
- 2000::/3    全球单播地址
- FE80::/10   链路本地地址
- FF00::/8    组播地址
- ::1/128     环回地址
```

## 9. 网络通信原理

### HTTP 通信流程

```
1. DNS 解析 → 获取域名对应 IP
2. TCP 三次握手 → 建立连接
3. 发送 HTTP 请求 → GET /index.html
4. 服务器处理请求 → 返回响应
5. TCP 四次挥手 → 关闭连接
```

### DNS 解析流程

```
1. 浏览器检查自身缓存
2. 检查系统缓存（hosts文件）
3. 查询本地DNS服务器
4. 若无，查询根域名服务器
5. 根域名服务器返回 .com 服务器
6. 查询 .com 域名服务器
7. 返回对应域名的 DNS 服务器
8. 查询该 DNS 服务器
9. 返回 IP 地址
```

### ARP 解析过程

```
ARP 请求（广播）:
"Who is 192.168.1.20? Tell AA:AA:AA:AA:AA:AA"
目标MAC: FF:FF:FF:FF:FF:FF（广播）

ARP 响应（单播）:
"192.168.1.20 is at BB:BB:BB:BB:BB:BB"
```

## 10. 常见网络问题与排查

### 常用诊断命令

#### ping 命令

```bash
# 测试连通性
ping www.example.com

# 持续 ping（Ctrl+C 停止）
ping -t www.example.com

# 指定次数
ping -n 10 www.example.com
```

#### tracert / traceroute 命令

```bash
# Windows 追踪路由
tracert www.example.com

# Linux/Mac 追踪路由
traceroute www.example.com

# 跳过 DNS 反向解析
traceroute -n www.example.com
```

#### ipconfig / ifconfig 命令

```bash
# Windows 查看 IP 配置
ipconfig
ipconfig /all

# Windows 刷新 DNS
ipconfig /flushdns

# Linux/Mac 查看 IP 配置
ifconfig
ip addr show
```

#### netstat 命令

```bash
# 查看网络连接
netstat -an

# 查看监听端口
netstat -an | grep LISTEN

# 查看路由表
netstat -r

# Linux 查看进程对应连接
netstat -tunlp
```

### 常见问题与解决方案

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 无法上网 | 网线未插好 | 检查网线连接 |
| 无法上网 | IP 配置错误 | 重新获取 IP（DHCP） |
| 无法上网 | DNS 故障 | 更换 DNS 服务器 |
| 无法访问特定网站 | 防火墙拦截 | 检查防火墙规则 |
| 网速慢 | 带宽不足 | 升级带宽 |
| 网速慢 | 网络拥塞 | 避开高峰时段 |
| 频繁掉线 | 信号干扰（WiFi） | 更换信道 |
| IP 冲突 | 多设备同 IP | 重新分配 IP |

### 网络排查流程

```
1. 检查物理连接
   │
   ▼
2. 检查 IP 配置
   │
   ▼
3. ping 本地网关
   │    │
   │    ├─ 成功 → 继续步骤4
   │    └─ 失败 → 检查网关配置
   │
   ▼
4. ping 外部 IP（如 8.8.8.8）
   │    │
   │    ├─ 成功 → 检查 DNS 配置
   │    └─ 失败 → 检查路由/ISP
   │
   ▼
5. ping 域名（如 google.com）
   │    │
   │    ├─ 成功 → 网络正常
   │    └─ 失败 → DNS 故障
   │
   ▼
6. 完成诊断
```

1. **网络类型**：PAN、LAN、MAN、WAN、GAN 按范围分类

6. **IP 地址**：32 位 IPv4，可划分为公网和私有
7. **端口套接字**：端到端通信的标识
8. **通信原理**：数据封装与解封装过程

掌握这些基础知识，是进行网络故障排查、系统设计和安全防护的前提。
