# Prompt Engineering 提示工程指南

> Prompt Engineering 是与 AI 模型高效沟通的核心技能，掌握它能显著提升输出质量。

## 一、为什么 Prompt Engineering 重要

同样的模型，不同的 Prompt 可能产生天壤之别的输出。好的 Prompt 能让：
- 输出更准确、更相关
- 减少幻觉和错误
- 获得更结构化的结果
- 让模型执行复杂任务

## 二、基础结构

### 2.1 Prompt 四要素

```
┌─────────────────────────────────┐
│  角色（Role）                     │  ← 你是谁
│  "你是一位资深全栈工程师"           │
├─────────────────────────────────┤
│  背景（Context）                   │  ← 上下文信息
│  "我们正在开发一个电商后台系统"      │
├─────────────────────────────────┤
│  任务（Task）                     │  ← 具体要做什么
│  "请设计订单模块的数据库表结构"      │
├─────────────────────────────────┤
│  约束（Constraints）               │  ← 输出要求
│  "使用 Markdown 表格，包含字段注释"  │
└─────────────────────────────────┘
```

### 2.2 System Prompt vs User Prompt

```python
messages = [
    {"role": "system", "content": "你是一位 Python 专家，用中文回答，代码附注释"},
    {"role": "user", "content": "如何用 pandas 读取 Excel 文件？"}
]
```

- **System Prompt**：设定模型的角色和行为规范，贯穿整个对话
- **User Prompt**：具体的用户请求

## 三、核心技巧

### 3.1 写得具体，而非模糊

```
❌ "写个排序算法"
✅ "用 Python 写一个归并排序算法，处理包含100万个随机整数的列表，
    计算执行时间，并验证排序结果正确性。"
```

### 3.2 提供示例（Few-Shot）

```
请将以下英文翻译为中文：

英文: "Hello, how are you?"
中文: "你好，你怎么样？"

英文: "Thank you for your help."
中文: "谢谢你的帮助。"

英文: "I love programming."
中文:
```

示例数量建议：
- 简单任务：1-3 个示例
- 复杂任务：3-5 个示例
- 示例越多越好，但注意 Token 限制

### 3.3 分步思考（Chain of Thought）

```
❌ 直接问答案
"一辆车从A城出发去B城，前半程速度60km/h，后半程速度80km/h，
 全程平均速度是多少？"

✅ 引导分步思考
"一辆车从A城出发去B城，前半程速度60km/h，后半程速度80km/h。
请一步步思考：1) 设全程距离 2) 计算前后半程时间 3) 计算平均速度。
给出每一步的详细过程。"
```

**CoT 的关键触发词**：
- "请一步步思考"
- "Let's think step by step"
- "先分析，再得出结论"
- "请详细推理过程"

### 3.4 角色设定

```
"你是一位有10年经验的资深架构师，精通分布式系统设计。
请从可扩展性、高可用、一致性三个维度分析这个方案。"
```

好的角色设定包含：
- 专业领域（资深架构师）
- 经验水平（10年经验）
- 专长方向（分布式系统）
- 输出角度（可扩展性、高可用、一致性）

### 3.5 分隔符标注

用特殊符号明确区分指令和内容：

```
请总结以下文章的核心观点，用3个要点呈现。

---
[文章内容]
---

请用以下格式输出：
1. **要点一**: ...
2. **要点二**: ...
3. **要点三**: ...
```

常用分隔符：`---`、`###`、`"""`、`<tag></tag>`

## 四、高级技巧

### 4.1 思维树（Tree of Thought, ToT）

让模型探索多条推理路径，选择最优解：

```
请解决这道数学题。对于每一步：
1. 列出2-3种可能的推理方向
2. 评估每种方向的可行性
3. 选择最有希望的方向继续
4. 如果发现走不通，回溯到上一个分支

[题目内容]
```

### 4.2 自我反思（Self-Reflection）

让模型检查和修正自己的输出：

```
第一步：请给出你的初步答案。

第二步：请以评审者身份，检查你的答案是否存在以下问题：
- 逻辑漏洞
- 事实错误
- 遗漏的重要信息

第三步：基于检查结果，给出修正后的最终答案。
```

### 4.3 ReAct 模式

推理（Reasoning）与行动（Action）交替进行：

```
你需要回答用户问题。每一步你可以：
思考（Thought）：分析当前状况，规划下一步
行动（Action）：使用工具获取信息 [搜索/计算/查询]
观察（Observation）：查看行动结果

用户问题：2024年诺贝尔物理学奖获得者是谁？他的主要贡献是什么？

Thought: 我需要搜索最新的诺贝尔物理学奖信息
Action: 搜索 "2024 Nobel Prize Physics winner"
Observation: [搜索结果]

Thought: 根据搜索结果，我可以回答了...
```

### 4.4 结构化输出

要求模型以特定格式输出：

```
请分析以下代码的性能问题，以 JSON 格式输出：

```json
{
  "issues": [
    {
      "location": "代码位置",
      "problem": "问题描述",
      "severity": "high/medium/low",
      "suggestion": "优化建议"
    }
  ],
  "summary": "总体评价"
}
```

### 4.5 防止幻觉

```
请回答以下问题。规则：
1. 只回答你确定知道的内容
2. 如果不确定，明确说"我不确定"
3. 不要编造信息或做出猜测
4. 如果需要最新信息，请说明你的知识可能有截止日期
```

## 五、常见场景的 Prompt 模板

### 5.1 代码生成

```
你是一位精通 {language} 的高级工程师。

需求：
{需求描述}

技术栈：
{相关框架/库}

约束条件：
- 代码必须包含类型注解
- 处理边界情况
- 添加中文注释
- 遵循 {language} 最佳实践

请输出完整代码，并在代码后附上关键设计说明。
```

### 5.2 文章写作

```
请写一篇关于 {主题} 的文章。

要求：
- 目标读者：{读者画像}
- 风格：{专业/通俗/幽默}
- 篇幅：{约X字}
- 结构：引言 → {自定义章节} → 总结
- 包含：具体案例、数据支撑

避免：
- 空洞的套话
- 过度使用形容词
- 缺乏实质内容
```

### 5.3 数据分析

```
请分析以下数据：

{数据内容或描述}

分析维度：
1. 总体趋势
2. 异常值识别
3. 关键指标计算
4. 可能的原因分析
5. 行动建议

输出格式：
- 先给出结论摘要（3句话以内）
- 再展开详细分析
- 使用表格呈现关键数据
```

### 5.4 代码 Review

```
请审查以下 {language} 代码，从以下维度评估：

1. **正确性**：是否存在逻辑错误
2. **安全性**：是否存在安全漏洞
3. **性能**：是否存在性能瓶颈
4. **可读性**：命名、注释、结构
5. **最佳实践**：是否符合语言/框架规范

对于每个问题，请给出：
- 问题位置（行号）
- 严重程度（🔴高 🟡中 🟢低）
- 修复建议和代码示例

```{代码}```
```

## 六、Prompt 调试与优化

### 6.1 A/B 测试

对同一个任务写多个版本的 Prompt，对比输出质量：

```python
prompts = {
    "v1_simple": "解释什么是机器学习",
    "v2_detailed": "用通俗易懂的方式解释机器学习，举一个日常生活中的例子",
    "v3_expert": "你是一位AI教授，请解释机器学习的核心概念、主要流派和实际应用",
}
```

### 6.2 迭代优化流程

```
1. 写初始 Prompt
2. 测试输出，记录问题
3. 分析问题原因（模糊？缺少上下文？格式不对？）
4. 修改 Prompt
5. 重复 2-4 直到满意
```

### 6.3 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 输出不相关 | 任务不明确 | 更具体地描述任务 |
| 输出太长 | 没有限制篇幅 | 加上长度约束 |
| 输出太简短 | 没要求详细 | 要求展开说明 |
| 格式不对 | 没指定格式 | 给出输出模板/示例 |
| 幻觉 | 超出知识范围 | 加上"不确定就说不确定" |
| 逻辑混乱 | 任务太复杂 | 拆分为多步，使用 CoT |

## 七、Prompt 编排（用于 AI Agent）

### 7.1 多 Prompt 协作

复杂任务需要多个 Prompt 串联：

```
Step 1（分析 Prompt）：
"分析用户需求，提取关键信息，以结构化 JSON 输出"

Step 2（规划 Prompt）：
"根据分析结果，制定执行计划，拆分为子任务"

Step 3（执行 Prompt）：
"按计划执行第 N 个子任务"

Step 4（整合 Prompt）：
"汇总所有子任务的结果，生成最终回复"
```

### 7.2 动态 Prompt

根据上下文动态生成 Prompt：

```python
def build_prompt(context):
    return f"""
    当前状态：{context['status']}
    已完成步骤：{context['completed_steps']}
    用户目标：{context['goal']}
    
    请根据当前状态，决定下一步操作。
    可选操作：{context['available_actions']}
    
    输出 JSON 格式：
    {{"action": "...", "reason": "..."}}
    """
```

## 九、速查表

```
✅ 好的 Prompt：
  - 具体明确，不模糊
  - 提供足够的上下文
  - 包含输出格式要求
  - 有示例（Few-Shot）
  - 分步引导复杂任务
  - 有角色设定

❌ 不好的 Prompt：
  - "帮我写个xxx"（太模糊）
  - 没有约束条件
  - 一次性塞太多不相关的任务
  - 没有示例或格式要求
  - 假设模型知道你的意图
```
